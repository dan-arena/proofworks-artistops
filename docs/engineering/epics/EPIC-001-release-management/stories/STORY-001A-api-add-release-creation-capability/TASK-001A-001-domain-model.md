# Language-Neutral Domain Definitions

## Object: Release

Represents an operationally managed release effort.

### Fields

| Field | Type | Required | Owner | Notes |
|---|---|---:|---|---|
| id | unique identifier | yes | backend | generated by backend |
| title | text | yes | backend validated | required |
| releaseType | enum | yes | backend validated | approved ReleaseType only |
| targetReleaseDate | calendar date | yes | backend validated | required |
| lifecycleState | enum | yes | backend | initialized to Draft |
| operationalNotes | text | no | backend validated | optional |
| createdAt | timestamp | yes | backend | assigned during creation |
| updatedAt | timestamp | yes | backend | initialized during creation |

- Lifecycle transitions are not part of this Task.
- Cancellation behavior is not part of this Task.

# Behavioral Rules

## Rule: Backend Generates Release Identifier

The backend must generate the Release identifier.

Callers must not supply identifiers.

---

## Rule: Backend Initializes Lifecycle State

New Releases must initialize with:

```text
Draft
```

No caller-controlled lifecycle initialization is allowed.

---

## Rule: Backend Assigns Timestamps

The backend must assign:

- createdAt
- updatedAt

during Release initialization.

---

# Pseudocode Behavior

## Behavior: Initialize Release

```text
function createRelease(input):

    require input.title exists
    require input.releaseType is supported
    require input.targetReleaseDate exists

    release.id = generateUniqueIdentifier()

    release.title = normalize(input.title)

    release.releaseType = input.releaseType

    release.targetReleaseDate = input.targetReleaseDate

    release.lifecycleState = Draft

    release.operationalNotes = input.operationalNotes

    release.createdAt = currentTimestamp()

    release.updatedAt = release.createdAt

    return release
```

# Implementation Risk Notes

## Risk: Lifecycle Transition Leakage

Implementation may accidentally introduce transition methods such as:

```text
activate
markReady
release
cancel
```

Mitigation:

Do not implement lifecycle transitions.

Only initialize Draft state.

---

## Risk: Persistence Overreach

Implementation may drift into schema-heavy persistence modeling.

Mitigation:

Only implement the domain model required for release creation.

---

## Risk: API Contract Leakage

Implementation may accidentally include request/response contracts.

Mitigation:

API contracts belong to TASK-001A-002.

Keep this Task domain-only.
